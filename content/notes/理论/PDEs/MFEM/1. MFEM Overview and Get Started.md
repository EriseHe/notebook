# 1 #
## 1.1 Introduction to MFEM and Features

> **MFEM (Modular Finite Element Methods)**: a free, lightweight, scalable C++ library for finite element methods, developed at Lawrence Livermore National Laboratory.

- **Features**:
  - High-Order Finite Element Spaces  
    ![](https://cdn.mathpix.com/cropped/d684899b-ce8b-4274-ac75-d8bb80b66e0b-05.jpg?height=287&width=276&top_left_y=139&top_left_x=1142)
  - Flexible discretization on unstructured grids
  - Parallel, scalable, and GPU-ready
  - Integration with other open-source packages (PETSc, Hypre, SUNDIALS, etc.)
  - Open-source: available on GitHub under BSD license
### 1.1.1 Resources

For more information, check:

- Home: https://mfem.org/
- Getting-started: https://mfem.org/getting-started/
- Tutorial: https://mfem.org/tutorial/
- Gallery: https://mfem.org/gallery/
## 1.2 Environment Setup: Docker Containers

### 1.2.1 Docker Containerization Platform

Challenge: set up and maintain complex environments for open-source libraries.

### 1.2.2 Solution: Docker

> Docker is a platform designed to help developers build, share, and run container applications.

  ![|80x70](https://cdn.mathpix.com/cropped/d684899b-ce8b-4274-ac75-d8bb80b66e0b-07.jpg?height=350&width=409&top_left_y=211&top_left_x=1072)
  
- Pre-built: specific image includes all dependencies (MPI, Hypre, NETCDF, SUNDIALS, MFEM) and the required tutorials.
- Portability: run the exact same environment on any operating system (Windows, Mac, Linux).

### 1.2.3 Run Container and Access VS Code

For more information, see MFEM - Docker Tutorial.

Step 1: Pull the Image

```bash
docker pull ghcr.io/lmolin3/mfem-docker/cpu-tpls:vscode
```

Step 2: Start the Container

```bash
docker run --name mfem-tutorial \
    --volume <PATH-TO-FOLDER>:/home/shared/ \
    -p 8080:8080 \
    ghcr.io/lmolin3/mfem-docker/cpu-tpls:vscode
```

## 1.3 Access Your Environment

The web-based VS Code environment can be accessed at `localhost:8080`.

### 1.3.1 Setup Instructions

Step 1: Start the Environment

- Launch the Docker image
- Open VS Code at `http://localhost:8080`
- Open the integrated terminal: `Ctrl + Shift + P → Terminal: Attach to Session`

### 1.3.2 Step 2: Update the Code Base

```bash
cd /home/euler/source
git pull
```

### 1.3.3 Step 3: Configure and Compile

```bash
cd /home/euler/build
cmake ../source/ -DUSER_CONFIG=../cmake-config/user-parallel.cmake
make -j 20 tutorial_emory_01
```

## 1.4 Post-processing and Visualization: ParaView

### 1.4.1 ParaView

- Interactive 3D visualization
- Scalable: designed to analyze and visualize extremely large datasets (TB) on distributed and shared-memory parallel computing systems
- Open-source and multi-platform: free to use and runs natively on Windows, Mac, and Linux, ensuring portability
- Extensibility (custom filters): supports scripting for automating tasks and creating custom data processing filters
- Alternatives: VisIt, GLVis
